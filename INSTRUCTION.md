# Інструкція з розгортання ToDo App

Цей репозиторій містить маніфести для розгортання додатка в кластері Kubernetes з налаштованим автоскейлінгом.

## Як розгорнути

1. Створіть неймспейс (якщо він не створений):
   `kubectl create namespace mateapp`
2. Застосуйте деплоймент:
   `kubectl apply -f deployment.yml`
3. Застосуйте автоскейлер:
   `kubectl apply -f hpa.yml`

## Пояснення конфігурації

### Resource Requests & Limits
- **Requests (CPU: 250m, Mem: 64Mi):** Вибрані на основі очікуваного мінімального споживання додатком у стані спокою, щоб гарантувати резервування потужностей у кластері.
- **Limits (CPU: 500m, Mem: 128Mi):** Обмежують споживання, щоб один под не міг "захопити" всі ресурси ноди.

### RollingUpdate Strategy
Використано `maxUnavailable: 1` та `maxSurge: 1`. Це дозволяє:
- Завжди мати хоча б один робочий под під час оновлення.
- Тимчасово створювати додатковий под для плавного перемикання трафіку без простоїв.

### HPA Configuration
- **Min/Max Replicas (2/5):** Гарантує високу доступність (мінімум 2 поди в різних зонах/нодах) та можливість масштабування до 5 подів при пікових навантаженнях.
- **Threshold (70%):** Баланс між ефективним використанням ресурсів та запасом потужності для обробки раптових сплесків трафіку.

## Як отримати доступ
Після деплою можна використати порт-форвардинг:
`kubectl port-forward deployment/todoapp 8080:8080 -n mateapp`
Додаток буде доступний за адресою `http://localhost:8080`.
